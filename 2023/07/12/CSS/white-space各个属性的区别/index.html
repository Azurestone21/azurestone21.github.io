CSS `white-space` 各个属性的区别

在 CSS 中，white-space 属性用于控制元素内空白字符（空格、制表符 \t、换行符 \n 等）的处理方式，以及文本是否会在容器边界自动换行。它直接影响文本的排版和可读性，是前端布局中处理文本格式的核心属性之一。

1. white-space: normal（正常处理）

默认属性，完全遵循浏览器对文本的“常规渲染规则”。

  • 合并空白字符：多个连续的空格、制表符会被合并为 1 个空格；文本开头/结尾的空白字符会被忽略（除非被其他元素包裹）。
  • 自动换行：当文本长度超过容器宽度时，会在单词边界（或标点符号后）自动换行，避免文本溢出容器。

2. white-space: nowrap（不自动换行）

禁止文本自动换行，但仍会合并空白字符。

  • 合并空白字符：与 normal 一致，连续空格、制表符合并为 1 个，首尾空白忽略。
  • 禁止自动换行：文本会强制在同一行显示，即使超过容器宽度，也会溢出容器（除非配合 overflow 属性处理）。

使用场景：

  1. 需要文本保持在一行显示
  2. 超出容器宽度自动省略（需配合 overflow: hidden 和 text-overflow: ellipsis）。

3. white-space: pre（保留空白，不自动换行）

“pre” 是“preserve”（保留）的缩写，该属性值会完全保留源代码中的空白字符，但禁止自动换行。

  • 保留空白字符：多个连续空格、制表符 \t、手动换行 \n 都会原样显示，不合并、不忽略。
  • 禁止自动换行：文本长度超过容器宽度时，不会自动换行，会横向溢出。

使用场景：

  • 显示代码片段（保留代码的缩进和换行）；
  • 显示需要精确格式的文本（如诗歌、表格）。

4. white-space: pre-wrap（保留空白，自动换行）

“pre-wrap” 是“preserve + wrap”的组合，兼具 pre 保留空白的特性和 normal 自动换行的特性，是处理“需保留格式且需自适应容器”文本的最佳选择。

  • 保留空白字符：连续空格、制表符、手动换行原样显示（与 pre 一致）。
  • 自动换行：文本长度超过容器宽度时，会在适当位置自动换行（与 normal 一致），避免溢出。

使用场景：

  • 显示用户输入的富文本（如评论、文章，需保留用户手动换行）；
  • 显示代码片段（需保留缩进，同时避免横向溢出）。

5. white-space: pre-line（合并空白，保留换行，自动换行）

“pre-line” 是“preserve line breaks”的缩写，仅保留**手动换行 **\n，但会合并其他空白字符，同时支持自动换行。

  • 部分保留空白：仅保留手动换行 \n，连续空格、制表符会被合并为 1 个（与 normal 一致）。
  • 自动换行：文本超过容器宽度时自动换行（与 normal 一致）。

使用场景：

  • 显示用户输入的文本（如博客评论，用户手动换行需保留，但多余空格可合并）；
  • 显示段落文本（需区分段落换行，同时避免多空格影响排版）。

6. white-space: break-spaces（增强版 pre-wrap，强制换行）

break-spaces 是 CSS3 新增的属性值，行为与 pre-wrap 类似，但对“空白字符的换行规则”更严格，会强制将过长的空白字符（如连续空格）也纳入自动换行范围。

  • 保留所有空白字符：与 pre-wrap 一致，空格、制表符、换行符均保留。
  • 强制自动换行：不仅文本内容会换行，连续的空白字符（如多个空格） 如果超过容器宽度，也会被拆分成多行（pre-wrap 不会拆分空白字符，可能导致空白部分溢出）。
  • 不压缩空白宽度：即使换行，空白字符的原始宽度也会保留（如制表符 \t 仍占固定宽度）。

:::info
与 pre-wrap 的关键区别：

当存在超长连续空白字符时，pre-wrap 会让空白字符溢出容器，而 break-spaces 会将空白字符拆分行显示。

:::

总结

  1. 常规文本排版：用 normal（默认），满足大多数场景；
  2. 禁止换行（如导航）：用 nowrap，配合 overflow: hidden 处理溢出；
  3. 保留格式不换行（如代码）：用 pre，适合固定宽度的代码块；
  4. 保留格式且自适应（如用户评论）：用 pre-wrap，兼顾格式与可读性；
  5. 仅保留换行（如段落）：用 pre-line，避免多余空格影响排版；
  6. 严格处理超长空白（如特殊格式文本）：用 break-spaces，适合对空白排版要求极高的场景。
