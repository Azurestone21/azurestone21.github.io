汇总微信小程序开发遇到的坑

一、生命周期

  1. 小程序热启动（后台进入），app.js 里面 onLaunch 获取到的场景值不正确，需要在 onShow 里面获取。

二、API

定位

Error: MiniProgramError {“errMsg”:”getFuzzyLocation:fail the api need to be declared in the requiredPrivateInfos field in app.json/ext.json”}

  • 解决：app.json 配置 requiredPrivateInfos: ['getFuzzyLocation']

路由

{“errMsg”:”navigateTo:fail APP-SERVICE-SDK:Unknown URL protocol "https://"“}

  • 原因：navigateTo 不能跳转 http/https 链接
  • 解决：使用 webview 组件显示

三、组件

input

  • 问题：使用 type="nickname"，onInput函数开发工具控制台获取不到值，预览可以

image

微信小程序富文本中GIF图片只会播放一次

  • 解决：使用Mphtml插件显示富文本，在微信开发者工具里面可能还是会只播放一次，但是在手机是可以重复播放的。

图片设置mode=”heightFix”，图片会伸缩一下，再显示正常。

  • 解决：设置图片的 height: 100%; width: auto；如果 mode 是 widthFix，width: 100%; height: auto。

scroll-view

在Dom结构发生变化会自动回滚到顶部

  1. 避免同级节点操作。把弹窗放在<font style="color:rgb(0, 0, 0);">ScrollView</font>里面。
    1. 例如 关闭 <font style="color:rgb(0, 0, 0);">Modal</font> 后，<font style="color:rgb(0, 0, 0);">ScrollView</font> 会自动滚动会顶部
  2. 使用 <font style="color:rgb(0, 0, 0);">Block</font> 组件包裹<font style="color:rgb(0, 0, 0);">ScrollView</font>的同级节点。

scroll-view里面设置tab切换列表内容，切换时会先往下滚动一下

  • 原因：设置了scroll-with-animation="true"。
  • 解决：
      1. 去掉scroll-with-animation="true"
      1. 使用页面滚动，而不是 scroll-view 滚动。

video

立即播放时会先黑屏加载

  • 背景：视频本地缓存，video 加载完成后，即 loadedmetadata 事件完成后，开始播放时仍会先显示黑屏。
  • 原因

https://developers.weixin.qq.com/community/develop/doc/000424408c8af89413789b79e53400

  • 解决
    • **方法 1： **使用 poster 设置封面图，注意 video 的封面图需要网络图片，不能是本地图片，刚开始使用没注意看，使用了缓存本地的图片文件，导致没有效果。
    • **方法 2：**延迟视频播放（大约 1s），等视频完全加载好，再使用 VideoContext 控制视频播放。

web-view

IOS 真机，web-view显示H5，H5的安全区不生效

  • 背景：在微信小程序 web-view 页面显示 H5，H5 也是自己的项目，H5 中使用safe-area-inset-bottom占位底部安全区域。安卓手机和微信开发工具都没有问题，但是 IOS 手机在浏览器中也生效，但是在小程序 web-view 中不生效。
  • 原因：未知
  • 解决：在 web-view 的 url中把安全区域的高度传给 H5，在 H5 中处理。

四、样式

  1. 不要在公共组件写 page 样式，否则样式会引入到所有的页面。
