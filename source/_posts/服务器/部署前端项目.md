## 前端项目

服务器公网：8.138.20.29

### 步骤

#### 1.打包项目

```sh
npm run build
```

#### 2.把打包后的 dist 文件夹传到服务器上

最简单用 xftp

项目文件需要上传到 nginx 文件夹里面

#### 3.配置 nginx

```sh
vim /usr/local/nginx/conf/nginx.conf
```

```conf
server {
  listen       80;
  server_name  localhost;

  #charset koi8-r;

  #access_log  logs/host.access.log  main;

  location / {
      root   html;
      index  index.html index.htm;
  }

  # 新增
  location /blogs {
      alias  project/admin-vue; # 项目存放目录 /usr/local/nginx/project/admin-vue
      index  index.html index.htm;
  }
}
```

#### 4.访问

<http://8.138.20.29/blogs/index.html>

### 问题

> 访问项目 index.html 404

解决办法：

在vite.config.ts中设置：`base: "./"`，打包静态文件必须是根路径，否则放到服务器找不到静态资源（同理于webpack中的publicPath 的配置）

> 访问项目 assets 静态文件 404

解决办法：

路由模式history改成hash模式

```js
// vue2
const router = new VueRouter({
  mode:"hash",
  routes
})

// vue3
const router = createRouter({
  history: createWebHashHistory(process.env.BASE_URL),
  routes
})
```

## Express项目

### 步骤

#### 安装Node

#### 安装 pm2

```sh
npm install pm2 -g
```

#### 上传Express项目并安装依赖

#### 开放项目端口

1. 开启端口号：`firewall-cmd --zone=public --add-port=3000/tcp --permanent`
2. 重启防火墙：`firewall-cmd --reload`
3. 添加安全组 3000 端口

#### 启动

```sh
pm2 start ./bin/www
```

### 问题

显示找不到命令

```sh
[root@iZ7xvgqq72fzxpvqh6itdkZ ~]# pm2 -v
-bash: pm: 未找到命令
```

解决：给pm2程序添加一个软链接

`find / -name pm2` 找到pm2安装后的所在路径：`/root/.nvm/versions/node/v14.17.0/bin/pm2`

```sh
[root@iZ7xvgqq72fzxpvqh6itdkZ ~]# find / -name pm2
/root/.nvm/versions/node/v14.17.0/lib/node_modules/pm2
/root/.nvm/versions/node/v14.17.0/lib/node_modules/pm2/lib/templates/logrotate.d/pm2
/root/.nvm/versions/node/v14.17.0/lib/node_modules/pm2/pm2
/root/.nvm/versions/node/v14.17.0/lib/node_modules/pm2/bin/pm2
/root/.nvm/versions/node/v14.17.0/bin/pm2
```

接着执行 `ln -s /root/.nvm/versions/node/v14.17.0/bin/pm2 /usr/local/bin` 命令为pm2程序添加软链接。
其中 `/root/.nvm/versions/node/v14.17.0/bin/pm2` 就是上一步查找到pm2程序的所在路径，而 `/usr/local/bin` 是根据$PATH环境变量得到的路径，在这些目录下的程序可以在系统的任意位置直接调用执行，所以将软链接添加到此目录下
