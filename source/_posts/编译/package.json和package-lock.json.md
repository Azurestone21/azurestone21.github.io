`package.json` 和 `package-lock.json` 是 Node.js 项目中管理依赖的两个核心文件。

## **<font style="color:rgb(38, 38, 38);">package.json </font>**是什么

**作用：**项目的「配置清单」，记录项目基本信息和依赖管理的核心配置。项目必需文件，没有它无法通过 npm install 安装依赖。

**<font style="color:rgb(0, 0, 0) !important;">主要内容</font>**<font style="color:rgb(0, 0, 0);">：</font>

- <font style="color:rgb(0, 0, 0);">项目元信息：名称、版本、作者、许可证等</font>
- <font style="color:rgb(0, 0, 0);">依赖声明：</font>`<font style="color:rgb(0, 0, 0);">dependencies</font>`<font style="color:rgb(0, 0, 0);">（生产依赖）和</font><font style="color:rgb(0, 0, 0);"> </font>`<font style="color:rgb(0, 0, 0);">devDependencies</font>`<font style="color:rgb(0, 0, 0);">（开发依赖）</font>
- <font style="color:rgb(0, 0, 0);">脚本命令：</font>`<font style="color:rgb(0, 0, 0);">scripts</font>`<font style="color:rgb(0, 0, 0);"> </font><font style="color:rgb(0, 0, 0);">字段定义可执行命令（如</font><font style="color:rgb(0, 0, 0);"> </font>`<font style="color:rgb(0, 0, 0);">npm run build</font>`<font style="color:rgb(0, 0, 0);">）</font>
- <font style="color:rgb(0, 0, 0);">其他配置：入口文件、兼容 Node 版本、模块类型等</font>

**<font style="color:rgb(0, 0, 0);">生成 </font>**`**<font style="color:rgba(0, 0, 0, 0.85) !important;">package.json</font>**`**<font style="color:rgb(0, 0, 0);"></font>**

```bash
npm init
```

<font style="color:rgb(0, 0, 0);">执行该命令后，npm 会引导填写项目的基本信息（如名称、版本、描述、入口文件等），完成后自动生成 </font>`<font style="color:rgba(0, 0, 0, 0.85) !important;">package.json</font>`<font style="color:rgb(0, 0, 0);"> 文件。</font>

<font style="color:rgb(0, 0, 0);">如果想跳过交互提问，直接生成默认配置的 </font>`<font style="color:rgba(0, 0, 0, 0.85) !important;">package.json</font>`<font style="color:rgb(0, 0, 0);">，可以使用：</font>

```bash
npm init -y
# 或
npm init --yes
```

## <font style="color:rgb(0, 0, 0);">package-lock.json 是什么</font>

**作用：**「锁定依赖版本」的文件，由 npm 自动生成和维护。保证团队协作或部署时，所有人安装的依赖版本完全一致。

**<font style="color:rgb(0, 0, 0) !important;">主要内容：</font>**

- <font style="color:rgb(0, 0, 0);">记录项目中</font>**<font style="color:rgb(0, 0, 0) !important;">所有依赖（包括嵌套依赖）的精确版本号</font>**
- <font style="color:rgb(0, 0, 0);">依赖的安装来源（如 npm 仓库地址）</font>
- <font style="color:rgb(0, 0, 0);">依赖树结构和校验信息（确保文件完整性）</font>

**<font style="color:rgb(0, 0, 0);">生成 </font>**`**<font style="color:rgba(0, 0, 0, 0.85) !important;">package-lock.json</font>**`

<font style="color:rgb(0, 0, 0);"> </font>`<font style="color:rgba(0, 0, 0, 0.85) !important;">package-lock.json</font>`<font style="color:rgba(0, 0, 0, 0.85) !important;">不需要手动生成，在</font>`<font style="color:rgba(0, 0, 0, 0.85) !important;">npm install</font>`<font style="color:rgba(0, 0, 0, 0.85) !important;">或</font>`<font style="color:rgba(0, 0, 0, 0.85) !important;">npm update</font>`<font style="color:rgba(0, 0, 0, 0.85) !important;">后会自动生成。</font>

## 依赖版本

版本号根据**大版本.次要版本.小版本** 格式规定

通常用范围符号（^、~）表示允许的版本范围（如 ^1.2.3），而非精确版本。

- `^`（插入符号）
  - 表示允许更新到不改变主版本号的最新版本
  - 例如 ^1.2.3 允许安装 1.2.3 及以上，但小于 2.0.0 的版本（如 1.2.4、1.3.0 等）
  - 这是大多数情况下的默认选择，平衡了稳定性和获取 bug 修复的需求
- `~`（波浪号）
  - 表示允许更新到不改变主版本号和次版本号的最新版本
  - 例如 ~1.2.3 允许安装 1.2.3 及以上，但小于 1.3.0 的版本（如 1.2.4、1.2.5 等）
  - 比 ^ 更严格，只接受补丁级别的更新，适合对稳定性要求极高的场景
- `*`
  - 表示允许安装最新版本的依赖包，但可能会导致版本不兼容，慎用。
- <font style="color:rgba(0, 0, 0, 0.85);">空</font>
  - <font style="color:rgba(0, 0, 0, 0.85);">版本号前没有符号（如 </font>`<font style="color:rgba(0, 0, 0, 0.85);">1.2.3</font>`<font style="color:rgba(0, 0, 0, 0.85);">），则表示只能安装</font>**<font style="color:rgb(0, 0, 0) !important;">精确的指定版本</font>**<font style="color:rgba(0, 0, 0, 0.85);">。</font>

### <font style="color:rgba(0, 0, 0, 0.85);">如何安装依赖</font>

首次安装依赖时，npm 根据 package.json 的范围规则选择合适版本，同时生成 package-lock.json 锁定该版本。

后续安装时，npm 会优先遵循 package-lock.json 中的精确版本，而非重新解析 package.json 的范围。
