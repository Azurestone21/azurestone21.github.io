在 HTML 中，JavaScript 代码放置在 `<head>` 和 `<body>` 中的区别主要体现在**执行时机**和**对 DOM 的访问能力**上：

## 放在 `<head>` 中

- **执行时机**：在 HTML 文档加载解析 `<body>` 内容之前就会执行
- **特点**：

```javascript
// 等待整个文档加载完成后再执行
document.addEventListener("DOMContentLoaded", function () {
  // 操作DOM的代码
});
```

- 如果脚本需要操作 DOM 元素（如获取某个 `<div>`），会失败，因为此时 `<body>` 中的元素还未加载
- 通常需要配合 `DOMContentLoaded` 事件或 `defer`/`async` 属性使用：

## 放在 `<body>` 末尾

- **执行时机**：在 `<body>` 中的 HTML 元素加载解析完成后执行
- **特点**：
  - 可以直接访问和操作页面中的 DOM 元素，无需等待事件
  - 不会阻塞页面的渲染，用户体验更好
  - 是推荐的常规做法

## 关键区别总结

| 位置          | 执行时机           | DOM访问能力 | 阻塞页面渲染 |
| ------------- | ------------------ | ----------- | ------------ |
| `<head>`      | 文档加载早期       | 受限        | 会阻塞       |
| `<body>` 末尾 | 文档内容加载完成后 | 完全可用    | 基本不阻塞   |

大多数情况下，将 JavaScript 代码放在 `<body>` 标签的末尾，这样既能保证正常访问 DOM，又不会影响页面加载速度。如果必须放在 `<head>` 中，则需要使用适当的方法确保 DOM 已准备就绪。
