## 涉及 API

```javascript
// 获取设置API
wx.getSetting({ withSubscriptions: true });
// 订阅消息API
wx.requestSubscribeMessage(options);
```

## 订阅消息

- 唤起订阅消息弹窗

```javascript
wx.requestSubscribeMessage(options);
```

- 成功返回信息

```json
{
  "tmplId1": "accept", // 如果开启并同意
  "tmplId2": "reject", // 如果关闭或拒绝
  "errMsg": "requestSubscribeMessage:ok"
}
```

## 检查用户订阅消息状态

```javascript
wx.getSetting({ withSubscriptions: true });
```

- 首次订阅 或 没有勾选“总是保持以上选择”

```json
{
  "mainSwitch": true
}
```

![](https://azurestone21.oss-cn-guangzhou.aliyuncs.com/blogs/20260308001545568.png)

- 勾选“总是保持以上选择”，会出现对应订阅消息授权状态。
  - 状态值包括'`accept`、`reject`、`ban`、`filter`。
  - `accept`表示用户**同意**订阅该条id对应的模板消息。
  - `reject`表示用户**拒绝**订阅该条id对应的模板消息。
  - `ban`表示已**被后台封禁**。
  - `filter`表示该模板因为模板标题**同名**被后台**过滤**。

```json
{
  "itemSettings": {
    "tmplId1": "accept",
    "tmplId2": "accept"
  },
  "mainSwitch": true,
  "tmplId1": "accept",
  "tmplId2": "accept"
}
```

## 重新订阅消息

- 如果是消息类型是“<font style="color:rgb(0, 0, 0);">一次性订阅</font>”，触发的消息只会发送一次，如果需要再次发送，需要重新订阅。
  - <font style="color:#DF2A3F;">即能发多少次订阅消息，取决于允许申请订阅消息的次数。每订阅一次，订阅次数就会加 1。</font>
  - <font style="color:#DF2A3F;">理论上订阅消息和收到消息是 1 对 1 的，但是有一些特殊场景。</font>
    - <font style="color:#DF2A3F;">1 对 1 场景：例如 A 订阅了“活动报名审核结果”的消息，审核结果一般只有一个，A 就会收到一个消息。</font>
    - <font style="color:#DF2A3F;">1 对多场景：例如 A 订阅了“申请交换个人信息”消息，A 的订阅次数加 1，但是 B、C 都做出申请操作，预想 A 会收到两个消息通知，但是实际上只会收到一个，因为 A 订阅次数只有一次。如果 A 事先没有订阅消息，那就会完全收不到消息。</font>
- **如果勾选了“总是保持以上选择”后，无论是否开启订阅消息，是否拒绝/允许申请，之后重新调用requestSubscribeMessage 都不再弹出弹窗，但是接口也会返回 requestSubscribeMessage:ok。**
  - 开发版小程序如果需要**重新授权**，需要在**微信开发者工具-清缓存-清除模拟器缓存-清除授权数据**。
  - 线上版没有办法重新授权，只能在设置里选择 接受/拒绝 消息通知。
- **如果没有勾选“总是保持以上选择”，每次调用 API 都会弹出弹窗。**

## 如何关闭订阅消息

1. 在**微信-服务通知**里面，可以设置不再接受对应的通知。

![](https://azurestone21.oss-cn-guangzhou.aliyuncs.com/blogs/20260308001545565.png)

2. 在**小程序-右上角三点-设置-通知管理**里面，可以关闭接收通知。

![](https://azurestone21.oss-cn-guangzhou.aliyuncs.com/blogs/20260308001545567.png)
