## 提示词

### 提示词分类

- **系统提示词**
  - 定义：大模型角色定位 + 回复逻辑
  - 位置：在Agent的“人设与回复逻辑”中设置
  - 作用：持续影响整个会话的响应模式
- **用户提示词**
  - 定义：用户直接提出的具体指令或问题
  - 位置：在对话框中输入
  - 作用：指导模型执行特定任务

## 插件

> 以查询天气为例

### 创建插件

1. 在 扣子 -> 管理 -> 资源库 中，点击 `+资源`，选择 `插件` 类型。
2. 填写插件的名称、介绍、描述、图标；
3. 选择 `插件工具创建方式` 为 `云侧插件 - 在 Coze IDE 中创建`；
4. IDE 运行时，我选择 `Node.js`；
5. 点击 `确认`，创建插件。

进入插件后，点击 `在IDE中创建工具`，进入插件编辑页面。

1. 先 `添加工具`，填写工具的名称（get_weather）、介绍（查询天气），点击 `确认`，创建工具；
2. 创建后会看到工具的编辑页面，包括 `代码` 和 `元数据`。
   1. `代码` 中实现工具方法。
   2. `元数据` 中添加工具的 `输入参数` 和 `输出参数`，必须和 `代码` 中的 `handler` 方法的入参和返回参数一致，否则会报错。

#### 代码

下面是创建工具后给出的默认代码，需要注意这里**只能修改 `handler` 方法里面的逻辑**，不要修改其他代码，否则会报错。

代码注释也有说明：

1. 每个文件都需要导出一个名为 `handler` 的函数。此函数是工具的入口。
2. 对于输入参数，您可以通过 `input.xxx` 获取测试输入值。
3. `logger` 用于打印日志的日志记录器实例，由运行时注入。
4. 该函数的返回数据应与声明的输出参数相匹配。

```js
import { Args } from '@/runtime';
import { Input, Output } from "@/typings/get_weather/get_weather";

/**
  * Each file needs to export a function named `handler`. This function is the entrance to the Tool.
  * @param {Object} args.input - input parameters, you can get test input value by input.xxx.
  * @param {Object} args.logger - logger instance used to print logs, injected by runtime
  * @returns {*} The return data of the function, which should match the declared output parameters.
  *
  * Remember to fill in input/output in Metadata, it helps LLM to recognize and use tool.
  */
export async function handler({ input, logger }: Args<Input>): Promise<Output> {
  return {
    message: "Hello, world!",
  };
};
```

**实现查询天气的逻辑**

```js
import { Args } from '@/runtime';
import { Input, Output } from "@/typings/get_weather/get_weather";
const axios = require('axios');

/**
  * Each file needs to export a function named `handler`. This function is the entrance to the Tool.
  * @param {Object} args.input - input parameters, you can get test input value by input.xxx.
  * @param {Object} args.logger - logger instance used to print logs, injected by runtime
  * @returns {*} The return data of the function, which should match the declared output parameters.
  *
  * Remember to fill in input/output in Metadata, it helps LLM to recognize and use tool.
  */
export async function handler({ input, logger }: Args<Input>): Promise<Output> {
  try {
    const { city } = input;
    if (city && city.trim()) {
      const url = `https://wttr.in/${encodeURIComponent(city)}?format=j1`;
      const res = await axios.get(url, { timeout: 8000 });

      if (res.data?.current_condition?.length === 0) {
        return {};
      }

      const current = res.data.current_condition[0];
      const area = res.data.nearest_area[0];
      const areaName = area.areaName[0].value;
      const country = area.country[0].value;

      return {
        weatherDesc: current.weatherDesc[0].value,
        tempC: current.temp_C,
        feelsLikeC: current.FeelsLikeC,
        windDirection: current.winddir16Point,
        windSpeed: current.windspeedKmph,
        humidity: current.humidity,
        pressure: current.pressure,
        observationTime: current.observation_time
      };

    } else {
      return {};
    }
  } catch (error) {
    return {};
  }
};
```

在上面的代码中：

1. `input` 接收用户输入的 `city` 参数。
2. 返回参数 `weatherDesc`、`tempC`、`feelsLikeC`、`windDirection`、`windSpeed`、`humidity`、`pressure`、`observationTime`。

#### 元数据

- 在 **输入参数** 中添加 `city` 参数。
- 在 **输出参数** 中添加 `weatherDesc`、`tempC`、`feelsLikeC`、`windDirection`、`windSpeed`、`humidity`、`pressure`、`observationTime` 参数，最好去掉 必填 勾选，可以返回空值，避免报错。

### 使用插件

先在提示词中写入 “你是一个天气助手，为用户提供城市的天气信息”。

在没有插件的情况下，询问天气，模型并不能直接获取天气信息。

```markdown
用户：获取广州今天的天气。

Agent：
很抱歉，目前暂时无法获取广州今日的天气数据。你可以尝试通过以下方式查询：
打开手机自带的天气应用，定位到广州市即可查看实时天气及预报；
在微信或支付宝中搜索 “天气” 小程序，输入 “广州” 获取详细天气信息；
访问中国天气网官方网站（http://www.weather.com.cn），精准查询广州的天气情况。
如果之后需要查询其他城市或日期的天气，欢迎随时和我说～
```

**进入智能体 -> 编排 -> 技能 -> 插件 -> 添加插件 -> 选择 “资源库工具” -> 选择创建好的插件。**

```markdown
用户：获取广州今天的天气。

Agent：
广州市当前的天气状况如下：湿度为 78%，风速是 17（单位未明确），气压为 1013（单位未明确），风向为南东南（SSE），天气状况为晴朗（Clear），当前温度是 24°C，体感温度为 26°C，观测时间是下午 4 点。
```

运行时会自动调用插件，获取天气信息。

## 知识库

### Coze 平台实操步骤

1. **进入资源库**：Coze → 左侧菜单 → 资源库
2. **创建知识库**：资源 → 知识库 → 命名
3. **上传文件**：拖拽/上传文件，支持批量上传
4. **文档切块**：自动切块，通常设置为300/500字
5. **向量化预处理**：分段预处理，生成向量
6. **查看结果**：预览文本处理效果
