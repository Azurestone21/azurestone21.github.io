## 一、AI Agent 核心概念

### 1. 定义

AI Agent = **AI管家**，是能够**感知环境、自主决策、使用工具完成任务**的智能实体。

### 2. 核心能力

- **自主性**：独立思考和决策
- **工具使用**：调用各种外部能力（如搜索、API、插件）
- **任务规划**：拆解并执行复杂任务

### 3. 核心架构

- **User Request**：用户发起请求
- **Agent**：核心大脑，统筹调度
- **Tools**：外部工具集
- **Memory**：记忆模块，存储上下文
- **Planning**：任务规划模块，分解复杂任务

### 4. 与传统大模型（如ChatGPT）的区别

| 对比维度 | 传统大模型   | AI Agent               |
| -------- | ------------ | ---------------------- |
| 定位     | 被动响应     | 主动执行               |
| 信息获取 | 仅限训练数据 | 可实时搜索/检索知识库  |
| 能力边界 | 只能对话     | 可调用工具、执行操作   |
| 任务处理 | 单轮回答     | 多步规划、复杂任务拆解 |
| 应用场景 | 咨询问答     | 实际业务流程自动化     |

## 二、提示词（Prompt）设计

### 1. 定义

提示词 = 与AI沟通的**说明书**，是开发者赋予Bot的身份、能力和行为规范，决定了Bot的响应质量和风格。

- 帮助用户控制语言模型输出，适配特定需求
- 对措辞和设计敏感，需要精心制定准则

### 2. 设计四要素

- **角色定位**：明确Bot身份，建立专业形象
- **技能描述**：清晰目标，让Bot知道要做什么
- **输出格式**：结构化回复要求，确保输出规范
- **约束条件**：限制不当行为，保证安全合规

## 三、RAG 技术与知识库构建

### 1. RAG 基础原理

RAG（Retrieval-Augmented Generation）是**结合知识检索和语言生成**的AI技术，主要用于解决大模型**幻觉问题**。

#### 传统LLM的缺陷

- 知识滞后：无法获取训练数据之外的最新知识
- 容易产生幻觉：编造不存在的信息

#### RAG 核心流程

1. **Retrieve（检索）**：将用户问题（Query）向量化后，在知识库/向量数据库中检索相关上下文（Context）
2. **Augment（增强）**：将原始问题与检索到的上下文融合，生成新的Prompt
3. **Generate（生成）**：将融合后的Prompt输入大模型，基于检索内容生成最终回答

### 2. RAG 知识库构建

#### 核心流程

**文档准备 → 文档切分 → 文档向量化**

#### 关键问题解答

- **文档为什么要切片？**
  为了适应大语言模型的上下文长度限制，并提升检索的精确度和效率。
- **文档向量化原因？**
  将文本进行向量数字化，便于计算问题和文档的相似性，实现高效语义检索。

## 四、Function Call（函数调用）

### 1. 定义

2023年6月13日 OpenAI 公布的功能，指在语言模型中集成**外部功能或API的调用能力**，让模型在生成文本过程中可调用外部函数/服务，获取额外数据或执行特定任务。

### 2. 应用基本流程（简化）

1. 输入用户请求到大语言模型
2. 模型判断**是否需要调用外部信息**
   - **是**：
     1. 匹配外部函数
     2. 选择合适的外部API
     3. 根据API逻辑生成回复
   - **否**：直接生成文本回复
3. 输出最终结果

## 五、工作流体系

### 1. 两种工作流类型对比

| 维度     | 工作流（Workflow）           | 对话流（Chatflow）         |
| -------- | ---------------------------- | -------------------------- |
| 适用场景 | 功能类、数据处理             | 对话类、交互式应用         |
| 典型案例 | 报告生成、海报制作、数据分析 | 智能客服、AI助手、虚拟伴侣 |
| 上下文   | 不支持对话历史               | 支持读取对话历史           |
| 会话管理 | 无会话概念                   | 必须传入会话名称           |
| 发布渠道 | API、模板、商店              | 全渠道（含社交平台）       |

## 六、总结

1. **AI Agent**：感知环境、自主决策、使用工具完成任务的智能实体，是“能干活的助手”
2. **Agent vs 大模型**：大模型是“聊天对象”，Agent不仅能聊，还能主动执行任务
3. **提示词**：搭建智能体的第一步，核心是明确角色、技能、格式与约束
4. **RAG**：通过“检索+生成”解决大模型幻觉与知识滞后问题，知识库构建核心是“文档切分+向量化”
5. **Function Call**：让模型具备调用外部工具/API的能力，突破自身能力边界
6. **工作流**：
   - **Workflow**：适合一次性数据处理、自动化任务
   - **Chatflow**：适合多轮对话、交互式场景
